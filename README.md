## JavaScript общие сведения

Это скриптовый язык - он не может существовать сам по себе. Это язык который встраивается, что характерно практически для всех скриптовых языков. У языка нет своего собственного ввода/вывода. Сам по себе, язык, не может получить откуда то данные и эти данные куда то вывести.

### Host среда

Место куда встраивается JavaScript согласно спецификации, называется Host - средой. Например браузер - это хост. Runtime (Agent по спецификации)- это встроенный язык в хост. Т.е внутрь хост среды встроен агент который может выполнять код языка JavaScript. Наиболее распространенные  и известные  Runtime среды это V8, SpiderMonkey, javaScriptCore. 
Большая часть того, чем оперирует JavaScript программист, является внешним api, некоторыми возможностями, которые предоставляет хост среда, это не документировано в спецификации. Важно дифференцировать ,  где перед нами непосредственно работа JavaScript и где результат работы внешнего Api. 
Например хост среда бараузе, предоставляет языку JavaScript внешнее Api вида DOM, EventLoop, CSSOM, все что связано с DOM, то что позволяет модифицировать и управлять DOM. Все это к языку JavaScript отношения не имеет, все это описывается и регламентируется некоторой отдельной спецификацией по которой работает хост среда. 
Типичным примером вышесказанного является Api SetTimeout. Мы можем использовать некоторый метод SetTimeout, который присутствует в глобальном объекте, он позволяет связать с некоторым истекающим таймером коллбэк, который должен быть вызван не ранее того времени , которое было передано этому апи. В хост среде браузера это апи подчинено стандарту HTML5 и описывается его спецификацией.Наряду с этим в другой хост среде существует точно такое же Api, с точно таким же названием SetTimeout, которое выполняет те же самые задачи, однако выполняется в соответствии с тем как хотелось людям которые разрабатывали эту хост среду. Например в браузере при выполнении вышеуказанного апи, мы получаем идентификатор, который впоследствии можем использовать для отмены выполнения запланированной задачи, а в хост среде NodeJs мы получаем инстанс объекта с методами, один из которых делает то же самое.
Структурирование информации.
Это концептуальный принцип который заложен в большое количество фундаментальных вещей, связанных с этим языком. Принцип этот чрезвычайно прост
Любая структуризация информации представляет собой список типа ключ -  значение. Где ключ для нас это то что позволяет нам найти, где находится то самое необходимое нам значение.
Одновременно существует возможность вместо конкретного значения, написать новый список, обозначив его фигурными скобками {}
В рамках этой концепции существует возможность определения ключей, которые мы называем служебные и эти ключи описываются и обозначаются включением в двойные квадратные скобки. Если мы встречаем у объекта такой вид записи ключа , то это значит что мы имеем дело со служебным свойством которое обязательно описано  в рамках спецификации JavaScript. Обратится к этому ключу мы не можем, оно скрыто. Он информационные.
Каждый условный список , имеет одну единственную обратную связь. Что это такое? Это когда вы имея какой то список у себя, обратились к ключу который в данном списке не существует, то происходит магия при которой без каких либо лишних действий, происходит обращение к ключу , который заключен в квадратные скобки. Внутри этого ключа описан адрес другого списка, и в том случае когда искомый ключ в нашем текущем списке не найден, происходит автоматический поиск в другом списке, который имеет с нашим списком связь при помощи [[ ]]нотации. При этом каждый список имеет одну единственную такую обратную связь. Соответственно если список полученный по обратной связи тоже не имеет искомого ключа, то он передаст поиск дальше в следующий список, если такой имеется, так как обратной связи может и не быть. При этом невозможно создать циклическую связанность, JavaScript это отслеживает и сигнализирует.
Кроме того существует возможность изменить обратную связь, назначив адресатом другой список
Суммируем. Существует общий принцип представления информации в JavaScript  -  любая информация в нем представляется в виде ключ - это нечто , что позволяет нам найти требуемое нам значение, при этом само значение может оказаться другим списком, с таким же ключом - значением. Оформляются такие списки фигурными скобками. Кроме того существует возможность изменить обратную связь, назначив адресатом другой список

Эти принципы важно понимать, так как многое в JavaScript подчинено этим принципам.

Выражения.

Любой символ и валидное выражение - это выражение для интерпретатора, выражение которое немедленно будет исполнено;
Например: исполнение в консоли кода равному 5 вернет 5, 5 + 7 вернет 12 .Почему? Как это происходит. Хост система передает на исполнение, агент парсит переданное и возвращает результат или ошибку.

var leftVal = 5
var rightVal = 7
1)leftVal + rightVal
2) 5 + 7

С точки зрения рантайма 5 + 7 - это команда 5 (не цифра) после которой существует некоторая синтаксическая конструкция + снова берем команду 7 (которая при выполнении какой то внутренней машинерии рантайме возвращает нам число 7) то же самое происходит при использовании идентификаторов. И так как что то делать с этим результатом дальше рантайм не может.Мы ему не дали никаких команд и распоряжений по этому поводу, то он просто возвращает нам результат .
